# Вимоги

Переказ завдання власними словами з явними критеріями приймання. (Оригінальний
бриф зберігається локально й свідомо не публікується в цьому публічному
репозиторії.)

## Контекст

Групу сайтів понизило Google, бо вони мають спільний **слід** (footprint)
теми — сприймаються як одна мережа (PBN). Завдання — зробити сайти візуально
**й** технічно несхожими, щоб вони більше не кластеризувалися як один оператор.
Це «унікалізація теми / усунення сліду».

## Частина 1 — Документ рішення (~1 год)

Виклад у вільній формі на 300–500 слів, що охоплює:

- що таке «унікалізація теми» з точки зору Google;
- які шари унікалізації існують і який найважливіший;
- як унікалізувати 5 сайтів за тиждень — що спершу, що наостанок;
- ризики й способи їх мінімізації.

Оцінюється структурованість мислення, а не єдина правильна відповідь.
→ [02-solution](02-solution.uk.md)

## Частина 2 — Реалізація (~3–4 год)

Обрати **одну** безкоштовну батьківську тему з wordpress.org (GeneratePress /
Astra / Kadence — на мій вибір, з обґрунтуванням) і зробити **дві дочірні теми**,
що відчуваються як **різні продукти**. Кожна має відрізнятися за:

| № | Вісь | Вимога |
|---|------|--------|
| 1 | **Колірна схема** | Повністю інша — не інший відтінок. |
| 2 | **Типографіка** | Різні пари шрифтів (заголовок + тіло). |
| 3 | **Хедер** | Інша структура (напр. центроване лого проти лого зліва + меню справа). |
| 4 | **Футер** | Інший копірайт **та** інша структура колонок. |
| 5 | **`style.css`** | Унікальні Theme Name / Author / Version, **без спільних коментарів чи підписів** між двома (це і є суть сліду). |
| 6 | **ACF-група полів** | Щонайменше одна кастомна ACF-група з **іншою структурою** в кожній темі й відмінною від батьківської. Зареєстрована в коді / закомічена як ACF Local JSON, щоб бути відтворюваною з репозиторію — а не лише кліками в БД. |

## Розгортання

Розгорнути **обидва** сайти на TasteWP або InstaWP (безкоштовний хостинг
WordPress) і додати живі посилання в README.

## Наскрізні обмеження

- **Ідіоматичний WordPress:** коректна структура дочірньої теми (заголовок
  `style.css` + `functions.php` з правильним enqueue батько→дитина),
  перевизначення шаблонів лише там, де потрібно, хуки/фільтри, ACF-групи,
  `wp_enqueue_*`, i18n та стандарти кодування WordPress.
- **Безпека:** екранувати весь вивід (`esc_html` / `esc_attr` / `esc_url`),
  санітизувати будь-який ввід, жодних секретів у репозиторії.
- **Якість фронтенду:** зовнішній CSS із багаторазовими класами, Flexbox,
  одиниці `rem` (мінімум `1rem`), семантичний HTML5, WCAG 2.1 AA.
- **Документація:** двомовна (англійська + українська), синхронізована.
- **noindex:** тримати розгорнуті демо поза індексом через налаштування сайту
  *Settings → Reading → Discourage search engines*, а **не** жорстко в коді
  дистрибутивної теми.
- **Git:** ініціалізувати тут; локальна ідентичність `alexnest-ua`
  <alexnest2002@gmail.com>; публічний репозиторій; покрокові conventional
  commits; без коміту секретів.

## Додаткові вимоги (розширений обсяг)

Запитано після початкового брифу й реалізовано в цьому репозиторії:

- Повний, унікально оформлений **набір шаблонів** на тему (`page/single/archive/
  search/404/index`) на **багаторазових `template-parts/`**.
- **Самохостингові шрифти** з `font-display: swap` (кращий CLS/LCP).
- Окремі теки **`css/` і `js/`** з **мініфікованими + несатисненими** збірками;
  підключаються лише мініфіковані (несатиснені — під `SCRIPT_DEBUG`).
- Найкращі практики **HTML5 / CSS / JS / доступності / SEO / семантики**.
- Професіоналізм WordPress через **окремий для кожного демо набір плагінів**
  та/або **власні `functions.php`**-еквіваленти (SEO-мета, очищення head).
- **Унікальний контент** і безплатні **зображення-плейсхолдери** на кожне демо.
- **Вичерпна двомовна документація** (цей набір `docs/`, EN + UK).

## Критерії приймання («готово»)

1. Дві дочірні теми **без спільного сліду**: відмінні заголовки `style.css`,
   кольори, шрифти, структура хедера й футера, текстові домени та префікси
   класів.
2. Одна відтворювана ACF-група на тему (Local JSON), що відрізняється
   структурою, під'єднана до фронтенду з екранованим виводом.
3. Обидві теми перевірені в реальному браузері (кольори, шрифти, хедер, футер,
   ACF), без PHP-повідомлень та помилок у консолі.
4. Обидва сайти розгорнуті й доступні; посилання в README; демо налаштовані на
   відмову від індексації.
5. Двомовна документація: вимоги, план, архітектура, база даних і рішення.
