припущеннями.

| Компонент | Версія / факт |
|-----------|---------------|
| PHP | 8.3.6 |
| База даних | MySQL 8.0.46 |

Локальна пісочниця ізольована від реальних проєктів на машині; PHP-FPM працює
від `alexnest`, тож теми можна розробляти в цьому репозиторії й під'єднувати
**симлінком** у пісочницю без `sudo` і без проблем із правами. WP-CLI додано в
домашню теку (без root) для встановлень і перевірки.

## Впорядкований план

1. **Ініціалізація репозиторію** — git із обов'язковою *локальною* ідентичністю,
   `.gitignore` (секрети, артефакти збірки, приватний бриф), `.editorconfig`
   (таби за WPCS).
2. **Налаштування dev** — встановити GeneratePress + ACF-free у пісочницю;
   створити демо-сторінки/записи, меню Primary і Footer, статичну головну.
3. **Прочитати батьківську тему** — намалювати карту хуків і фільтрів хедера/
   футера/навігації GeneratePress *перед* будь-яким перевизначенням
   (перевизначати лише потрібне). → [архітектура](04-architecture.uk.md)
4. **Дочірня тема A — Verdal** — заголовок `style.css` + токени, `functions.php`
   (enqueue, розташування навігації, точка завантаження ACF, рендер intro,
   кастомний футер), ACF Local JSON; симлінк, активація, перевірка, коміт.
5. **Дочірня тема B — Meridian Edge** — та сама поверхня, зроблена як свідомо
   **інший** продукт; симлінк, активація, перевірка, коміт.
6. **Перевірка в браузері** — відкрити обидві на `https://testwp`, підтвердити
   відмінні кольори / шрифти / хедер / футер і робочий ACF, перевірити консоль,
   зробити скриншоти; виправити те, що виявить тестування.
7. **Документація** — рішення Частини 1, вимоги, цей план, архітектура, база
   даних, рішення — усе двомовно; далі README.
8. **Розгортання** — обидва сайти на InstaWP, увімкнути *Discourage search
   engines*, додати живі посилання.
9. **Фінальний огляд** — код-рев'ю + перевірка безпеки, підтвердити, що демо
   відкриваються.

## Підхід до перевірки (планка приймання)

- **PHP:** `php -l` на кожному PHP-файлі; сторінки завантажуються з **нулем**
  повідомлень/попереджень (`curl` grep + браузер).
- **Структура:** `curl`-маркери для кожної теми (хендли enqueue, назви класів,
  тексти); перехресно перевірено, що маркери теми A **відсутні** на темі B
  (немає протікання сліду).
- **ACF:** `get_field()` повертає збережені значення на фронтенді (доводить, що
  Local JSON завантажився), banner/intro рендеряться з екранованим виводом.
- **Візуально:** скриншоти в реальному браузері на десктопній ширині для
  головної, футера й CTA одиночного запису обох тем.
- **Консоль:** без помилок/попереджень при завантаженні.

## Нотатки щодо обсягу

- Перша версія була мінімальною дочірньою темою GeneratePress лише на хуках. Потім
  обсяг розширено (на запит) до повного набору шаблонів, самохостингових шрифтів,
  конвеєра мініфікованих CSS/JS, SEO + очищення head і прогресивного JS — див.
  [рішення](06-decisions.uk.md).
- Досі **класична, не блокова**: GeneratePress — тема на PHP-шаблонах, тож дочірні
  лишаються класичними (без `theme.json`); без конструктора сторінок.
- **Хедер і футер лишаються на хуках**, попри перевизначення контент-шаблонів —
  дублюється лише те, що потрібно кожній темі.
- ACF використовує лише **безкоштовні** типи полів (без Pro repeater / flexible
  content).
