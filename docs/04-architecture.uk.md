# Архітектура та структура коду

## Структура репозиторію

```text
growica-test-task/
├── README.md / README.uk.md          # огляд + обґрунтування батьк. теми + встановлення + живі посилання
├── .editorconfig                      # таби для PHP/CSS/JS (стандарти кодування WordPress)
├── .gitignore                         # секрети, артефакти збірки, приватний бриф
├── docs/
│   ├── 01-requirements.{en,uk}.md
│   ├── 02-solution.{en,uk}.md         # есе Частини 1
│   ├── 03-implementation-plan.{en,uk}.md
│   ├── 04-architecture.{en,uk}.md     # (цей файл)
│   ├── 05-database.{en,uk}.md
│   └── 06-decisions.{en,uk}.md
└── themes/
    ├── verdal/                        # Дочірня тема A — редакційна / wellness
    │   ├── style.css                  # заголовок теми + повний стиль (токени + класи)
    │   ├── functions.php              # enqueue, розташування навігації, футер, ACF
    │   └── acf-json/
    │       └── group_verdal_page_intro.json
    └── meridian-edge/                 # Дочірня тема B — продукт / tech
        ├── style.css
        ├── functions.php
        └── acf-json/
            └── group_meridian_edge_cta.json
```

Теми встановлюються копіюванням однієї теки з `themes/` у `wp-content/themes/`.
Кожна — **класична** дочірня тема GeneratePress: без кроку збірки, без блокової
теми / `theme.json`.

## Анатомія дочірньої теми

| Файл | Призначення |
|------|-------------|
| `style.css` | Обов'язковий заголовок теми WordPress (унікальний для кожної) **плюс** повний стиль: токени дизайну в `:root`, багаторазові BEM-подібні класи, розкладка Flexbox, одиниці `rem`, focus-visible, reduced-motion. |
| `functions.php` | Налаштування теми (i18n, меню футера), enqueue ресурсів, розташування навігації, точка завантаження + рендер ACF, кастомний футер. Весь вивід екранований. |
| `acf-json/*.json` | ACF **Local JSON** — визначення групи полів, під версійним контролем, авто-завантаження. |

## Enqueue стилів батько → дитина (свідома деталь)

GeneratePress підключає власний CSS під хендлом **`generate-style`** (з
`assets/css/`, а *не* зі свого `style.css`). Тому дочірня тема **не** підключає
наосліп батьківський `style.css` (де немає CSS). Натомість вона підключає власний
стиль, оголошуючи батьківський хендл залежністю, щоб він завжди завантажувався
після батьківського:

```php
wp_enqueue_style( 'verdal-style', get_stylesheet_uri(), array( 'generate-style' ), VERDAL_VERSION );
```

Шрифти підключаються окремо (Google Fonts, `display=swap`) з підказками ресурсів
`preconnect` через фільтр `wp_resource_hints`.

## Використані хуки й фільтри (перевизначати лише потрібне)

| Аспект | Механізм | Verdal (A) | Meridian Edge (B) |
|--------|----------|------------|-------------------|
| Розташування навігації | фільтр `generate_navigation_location` | `nav-below-header` (центровано) | `nav-float-right` (лого зліва/меню справа) |
| Футер | `remove_action` футера GP + `add_action('generate_footer', …)` | 3 колонки + центрований копірайт | темний, 4 колонки + розділений нижній рядок |
| Час видалення хука футера | відкладено до `after_setup_theme` | ✔ | ✔ |
| Джерело ACF | `acf/settings/load_json` → `acf-json/` теми | ✔ | ✔ |
| Рендер ACF | `generate_after_entry_header` | intro сторінки | — |
| Рендер ACF | `generate_after_content` | — | CTA одиночного запису |
| Меню футера | `register_nav_menus('footer-menu')` | ✔ | ✔ |

> **Зауваження про порядок завантаження:** `functions.php` дочірньої теми
> завантажується *перед* батьківським. GeneratePress реєструє колбеки футера під
> час завантаження батьківської теми, тож виклики `remove_action()` відкладено
> до `after_setup_theme` — інакше вони спрацювали б зарано й нічого б не зробили.
> (Виявлено під час тестування в браузері; див. [рішення](06-decisions.uk.md).)

Файли шаблонів батьківської теми не копіюються — структура хедера, футера й
навігації досягається хуками/фільтрами + CSS, що тримає дочірні теми легкими, а
розходження — суто адитивним.

## Безпека виводу та доступність

- Кожне динамічне значення екранується на виводі: `esc_html`, `esc_attr`,
  `esc_url` та `esc_html__` / `printf` для перекладних рядків.
- Масиви link/image з ACF перевіряються на null перед використанням; зображення
  рендериться з `alt`, `width`/`height` (без CLS) і `loading="lazy"`.
- Семантичні орієнтири: один `<footer>` на сторінку, `<nav aria-label>` для меню
  футера, `<aside>` для CTA. Контури focus-visible, цілі дотику ≥44px на кнопках,
  палітри з контрастом AA та обробка `prefers-reduced-motion`.
